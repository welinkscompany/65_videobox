# Methodology

- Inspected the current VideoBox source and authoritative plans.
- Pinned every upstream repository to an observed commit.
- Split the investigation across SaaS shell, OpenCut, and Opencast/VideoBox mapping agents.
- Used only official repositories, official documentation, official deployments, and current local source for conclusions.
- Classified each candidate as locked owned source, partial port, attributed behavioral adaptation, reference only, or reject.
- Recorded 13 sources and 6 key claims.
- Ran `validate_ledger.py`; 6 claims verified, 0 unresolved, 0 refuted.
- Kept legal interpretation narrow: licenses and notice requirements were recorded, while ambiguous Supabase package metadata was treated conservatively as reference-only.
