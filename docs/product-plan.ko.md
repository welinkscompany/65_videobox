# VideoBox 제품 계획서

## 1. 제품 개요

VideoBox는 로컬 우선으로 실행되는 AI 영상 자동 편집 엔진이다.

이 제품의 핵심 목적은 사용자가 제공한 다음 입력을 분석해:

- 녹음 파일
- 참고 문서 또는 스크립트
- 원본 영상
- 개인 B-roll 자산

자동으로 1차 편집 초안을 만들고, 최종 수정은 CapCut에서 이어갈 수 있게 하는 것이다.

## 2. 제품 정체성

VideoBox는 다음이 아니다.

- 완전한 프리미어/다빈치 대체 편집기
- 처음부터 풀 NLE를 직접 만드는 프로젝트
- 완전 자동 최종본 생성기

VideoBox는 다음이다.

- 로컬 우선 AI 편집 엔진
- 자동 초안 생성기
- CapCut 연동 중심 편집 파이프라인
- 개인 자산 재사용이 강한 B-roll 기반 편집 시스템

## 3. 핵심 사용자 문제

### 3.1 나레이션/설명형 영상 제작이 느리다

사용자는 녹음 파일과 문서/스크립트를 주더라도:

- 어디에 어떤 B-roll을 넣을지
- 어떤 음악이 맞는지
- 어디를 자르고 붙일지
- 어떤 설명 이미지를 넣을지

하나하나 수작업으로 정해야 한다.

### 3.2 긴 원본 영상을 정리하는 데 시간이 많이 든다

긴 원본 영상에서:

- 쓸만한 구간 찾기
- 반복 take 제거
- 무음 제거
- 흔들리거나 너무 어두운 구간 제외

이 과정이 오래 걸린다.

### 3.3 가진 B-roll을 반복적으로 재사용하기 어렵다

개인 라이브러리에 있는 B-roll은 시간이 갈수록 자산이 되지만:

- 찾기 어렵고
- 태그가 일관되지 않고
- 프로젝트 맥락에 맞는 추천이 어렵다

## 4. v1 목표

v1의 목표는 `사람이 빠르게 검수할 수 있는 자동 편집 초안`을 만드는 것이다.

즉:

- 완벽한 최종본 자동 생성보다
- 사람이 수정하기 쉬운 높은 품질의 초안 생성

이 우선이다.

## 5. 핵심 워크플로우

1. 프로젝트 생성
2. 녹음 파일 또는 원본 영상 입력
3. 참고 문서/스크립트 입력
4. 사용자 B-roll 라이브러리 연결
5. STT 및 세그먼트 분석
6. 반복 take, 무음, 재시작 구간 탐지
7. 문장/장면별 B-roll 후보 추천
8. 음악 후보 추천
9. 설명형 비주얼 카드/이미지/간단 애니메이션 계획
10. timeline JSON 기반 자동 초안 생성
11. preview 렌더
12. CapCut export
13. 사람 검수 및 최종 수정

## 6. 제품 원칙

### 6.1 로컬 우선

무거운 영상 처리와 렌더링은 사용자의 컴퓨터에서 수행한다.

### 6.2 SaaS 확장 가능

초기 구현은 로컬 우선이지만:

- 코어 엔진
- 데이터 모델
- job 모델
- provider 인터페이스

는 나중에 SaaS로 확장 가능한 구조로 설계한다.

### 6.3 내부 기준은 timeline JSON

CapCut은 내부 기준 포맷이 아니라 export 대상이다.
내부 편집 의사결정의 기준은 반드시 timeline JSON이어야 한다.

### 6.4 추천과 자동 적용을 분리

초기에는 아래 항목을 무조건 자동 적용하지 않는다.

- B-roll 선택
- 컷 삭제
- 음악 배치
- 설명형 비주얼 삽입

가능한 경우:

- 추천 점수
- 자동 적용 가능 여부
- 검수 필요 플래그

를 함께 제공한다.

### 6.5 확신도와 이유를 보여준다

초안 생성 결과에는 가능하면 다음을 포함한다.

- 적합도 점수
- confidence
- 추천 이유
- 검수 필요 여부

## 7. v1 핵심 기능

- 로컬 프로젝트 생성
- 로컬 미디어 ingest
- 녹음 파일 STT
- 세그먼트/장면 분석
- 반복 take 및 무음 탐지
- B-roll 추천
- 기본 음악 추천
- 설명형 카드/이미지/간단 애니메이션 계획
- timeline JSON 생성
- preview 렌더
- CapCut export
- 얇은 review UI

## 8. v1에서 하지 않을 것

- 자체 풀 타임라인 편집기
- 완전 자동 최종본 보장
- 결제
- 멀티유저 협업
- 브라우저 기반 완전 SaaS 편집기
- 고급 생성형 비디오 제작
- 고급 음악 믹싱/마스터링

## 9. 대상 사용 방식

### 9.1 설명형 나레이션 영상

녹음 파일 + 참고 문서 + B-roll + 설명 카드 조합으로 영상 초안을 만든다.

### 9.2 긴 원본 영상 정리

긴 원본 영상을 자동 분석해 컷 후보와 러프컷 초안을 만든다.

### 9.3 숏폼 후보 추출

롱폼 초안 또는 전사 결과를 기반으로 shortform 후보를 추천한다.

## 10. 성공 기준

VideoBox v1은 다음이 가능하면 성공으로 본다.

- 사용자가 녹음 파일과 스크립트를 넣을 수 있다
- 시스템이 세그먼트 단위로 자막과 편집 초안을 만든다
- B-roll과 음악 후보를 자동 추천한다
- 설명형 비주얼 계획을 일부 자동 생성한다
- preview를 확인할 수 있다
- CapCut으로 내보내 후편집할 수 있다
- 사용자가 수작업 시간을 확실히 줄였다고 느낀다
