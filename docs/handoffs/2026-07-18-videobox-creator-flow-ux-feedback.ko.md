# VideoBox 제작 흐름 UX 피드백 메모

**Date:** 2026-07-18
**Status:** 사용자 현장 테스트에서 확인됨 — 다음 UX 보완 작업의 필수 입력

## 확인된 문제

1. 제작 시작·질문 화면의 여백, 버튼, 파일 선택, 설명 문구가 지나치게 붙어 있어 읽기와 다음 행동 선택이 답답하다.
   - 화면의 정보 우선순위, spacing scale, 입력 그룹과 보조 행동의 분리가 필요하다.
   - 색상만 조정하는 수준이 아니라 layout, typography, component density를 다시 설계한다.

2. 질문 1의 답변이 질문 2 입력란에 그대로 남는다.
   - 질문별 draft input state를 분리한다.
   - 새 질문은 해당 질문의 저장된 답만 복원하고, 아직 답하지 않은 질문은 빈 입력으로 시작한다.
   - 회귀: 1번 답 저장 → 2번 진입 시 1번 답이 보이지 않는다.

3. 질문 흐름에 `이전 질문`이 없다.
   - 이전 질문 이동을 제공한다.
   - 이전 답변은 보존·복원하며, 현재 단계/진행률도 일관되게 갱신한다.
   - 요약 화면에서는 모든 답변을 수정할 수 있어야 한다.

4. 마지막 `기획 요약`이 비어 있어 승인할 내용을 사용자가 확인할 수 없다.
   - 마지막 답변 저장 뒤, 대본과 질문별 저장 답변을 즉시 요약 입력란에 채운다.
   - 요약은 대상, 게시 위치, 원하는 행동 등 사용자가 답한 내용을 읽기 쉬운 문장으로 보여 준다.
   - 빈 요약에서는 `요약 승인`을 활성화하거나 초안 준비 단계로 진행하지 않는다.
   - 회귀: 세 질문을 저장 → 요약 화면 진입 시 각 답이 포함된 수정 가능한 요약이 보인다.

5. 모든 제작자-facing 문구는 유진의 짧고 행동 중심적인 안내 원칙을 유지한다.
   - 시스템/개발/내부 ID/언어 이름을 화면에 노출하지 않는다.

## 적용 원칙

- 위 2~3번은 시각 개선이 아니라 제작 흐름의 기능 회귀로 다룬다. 구현 전 RED 테스트를 먼저 추가한다.
- 위 1번은 다음 responsive UX/UI 개선 slice의 acceptance 기준으로 포함한다.
- 현재 Task 9 사람 승인 및 Task 10 closeout 상태는 변경하지 않는다.
